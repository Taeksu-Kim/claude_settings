## 접근 방식

### 현재 문제
- 3개 워크플로우 모두 결과 기록 템플릿이 `## 리뷰 사이클: N/3`으로 시작하는 단일 구조.
- 서브 에이전트가 매 사이클마다 파일을 새로 작성하면 이전 피드백이 유실된다.
- 다음 사이클의 서브 에이전트가 이전에 어떤 지적이 있었는지 알 수 없다.

### 변경 방향
각 워크플로우의 "결과 기록" 섹션에 다음 2가지를 추가한다:

1. **누적 기록 규칙**: 이전 사이클의 내용을 삭제하지 않고, 새 사이클을 하단에 추가하는 규칙을 명시.
2. **템플릿 구조 변경**: 파일 최상단에 제목(`# Plan Review - <task_id>` 등)을 한 번만 두고, 각 사이클을 `---` 구분선과 함께 누적 기록하는 형식으로 변경.

### 구체적 변경

**plan-review/workflow.md** (30~63행 부근):
- 기존 템플릿의 `# Plan Review - <task_id>` + `## 리뷰 사이클: N/3` 구조를 → `# Plan Review - <task_id>` 제목은 최초 1회만, 이후 `---` + `## 사이클 N/3` 섹션을 하단에 추가하는 형태로 변경
- "2회차 이상에서는 기존 내용을 유지하고 하단에 추가한다" 규칙 명시
- "이전 사이클의 피드백과 비교하여 해소 여부를 명시한다" 지침 추가

**code-review/workflow.md** (42~81행 부근):
- 동일한 누적 기록 규칙 적용 (제목: `# Code Review - <task_id>`)

**documentation/workflow.md** (68~107행 부근):
- 동일한 누적 기록 규칙 적용 (제목: `# Documentation Review - <task_id>`)

### 변경 후 템플릿 예시 (plan-review 기준)

3개 워크플로우 모두 동일한 패턴을 적용한다. plan-review를 대표 예시로 제시한다.

### 피드백

- 문제점: ...
- 개선방안: ...

### 종합 판정: **리플랜 필요**

- 2회차 이상에서는 "이전 사이클 피드백 해소 여부" 인용 블록을 체크리스트 앞에 배치
