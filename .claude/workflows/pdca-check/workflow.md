# PDCA-Check Workflow

## PDCA 전체 구조에서의 위치

본 워크플로우는 `code-work` 워크플로우의 PDCA 사이클 중 **Check 단계**를 담당한다.

```
code-work PDCA 사이클:
  1. Plan  → 요청 분석, 작업 범위 정의
  2. Do    → 실패하는 테스트 작성 (TDD Red)
  3. Check → ★ 본 워크플로우 (테스트 실행 + 코드 품질 점검 + 수정)
  4. Act   → Check 완료된 코드 기반으로 본 구현
```

산출물 경로: `claude_task/<task_id>/check/check.md`

## Check 수행 절차

### 1. 산출물 읽기
- `claude_task/<task_id>/plan/plan.md` — 작업 의도와 범위 파악
- `claude_task/<task_id>/do/do.md` — 테스트 목록과 검증 의도 파악
- Do 단계에서 작성한 테스트 파일 확인

### 2. 테스트 실행
- pytest로 테스트 실행 후 결과 확인

### 3. 코드 품질 점검
다음 항목을 점검한다:
- **메모리 효율성**: 불필요한 복사, 과도한 메모리 사용
- **변수명/함수명**: 의도가 명확하고 일관된 네이밍
- **오버 엔지니어링 방지**: 현재 불필요한 추상화나 기능
- **미사용 코드 삭제**: 사용되지 않는 import, 변수, 함수
- **근본적 문제 해결**: 증상이 아닌 올바른 설계와 근본 원인 해결

### 4. 수정
- 문제가 발견되면 직접 수정한다.

### 5. 결과 기록
`check.md`에 다음 형식으로 기록한다:

```markdown
# Check - <task_id>

## 테스트 실행 결과
- 실행 명령: `pytest path/to/test_file.py`
- 결과: PASSED / FAILED
- 실패 항목: (있는 경우 테스트명과 실패 사유)

## 체크리스트

### 코드 품질
- [ ] 메모리 효율성: 불필요한 복사, 과도한 메모리 사용 없음
- [ ] 변수명/함수명: 의도가 명확하고 일관된 네이밍
- [ ] 오버 엔지니어링 방지: 불필요한 추상화나 기능 없음
- [ ] 미사용 코드 삭제: 사용되지 않는 import, 변수, 함수 없음
- [ ] 근본적 문제 해결: 올바른 설계와 근본 원인 해결

## 수정 사항
(문제 발견 시 수정한 내용을 파일 경로와 함께 기록)
```

- 통과한 항목은 `[x]`, 미통과 항목은 `[ ]`로 표기
- 미통과 항목은 직접 수정 후 `[x]`로 갱신
