# Claude MD Review Workflow

현재 프로젝트의 `CLAUDE.md`를 `claude_docs/claude_md_guide.md`의 기준에 따라 검토하고,
문제점과 개선점을 제시한다.

## 산출물 경로

```
claude_output_docs/
└── claude_md_review.md    # 리뷰 결과 (누적 기록)
```

## 터미널 출력 규칙

각 스텝 진입 시 출력한다:
```
echo -e "\033[36m[CLAUDE-MD-REVIEW STEP N] <단계명>\033[0m"
```

## 수행 절차

### 1. 파일 읽기

다음 두 파일을 읽는다:
- `CLAUDE.md` — 검토 대상
- `claude_docs/claude_md_guide.md` — 평가 기준

### 2. 평가

가이드의 각 기준에 따라 현재 `CLAUDE.md`를 평가한다.

#### 포함 여부 점검 (가이드 섹션 4 기준)
- [ ] 기술 스택과 버전이 구체적으로 명시됨
- [ ] 프로젝트 구조와 주요 폴더 역할이 설명됨
- [ ] 빌드, 테스트, 실행 핵심 명령어가 포함됨
- [ ] 아키텍처 철학(WHY)이 설명됨
- [ ] 사용하는 패턴이 명시됨
- [ ] 사용하지 않는 패턴이 명시됨
- [ ] 도메인 용어 매핑이 포함됨
- [ ] 테스트 방법이 명시됨
- [ ] Git 워크플로우/브랜치 규칙이 포함됨
- [ ] Claude가 자주 틀리는 항목이 기록됨

#### 제외 여부 점검 (가이드 섹션 5 기준)
- [ ] API 키, 비밀번호, 연결 문자열이 없음
- [ ] Linter/Formatter가 처리하는 스타일 규칙이 없음
- [ ] 프레임워크 기본 지식 설명이 없음
- [ ] 과도한 문서/복붙 내용이 없음
- [ ] 프로젝트 역사/히스토리가 없음
- [ ] 특정 태스크에만 해당하는 지시사항이 없음

#### 길이 및 구조 점검 (가이드 섹션 6 기준)
- [ ] 300줄 이하 (이상적으로 50~100줄)
- [ ] 상세 내용은 파일 참조(pointer) 방식으로 분리됨
- [ ] @-import로 파일 직접 임베드하지 않음

#### Anti-Pattern 점검 (가이드 섹션 8 기준)
- [ ] 금지 플래그 명시 시 대안도 함께 제시됨
- [ ] 포괄적 매뉴얼 형태가 아닌 Claude가 자주 틀리는 것 중심으로 작성됨
- [ ] 스타일 규칙이 결정론적 도구(linter 등)로 위임됨

### 3. 결과 기록

`claude_output_docs/claude_md_review.md`에 아래 형식으로 기록한다.

#### 누적 기록 규칙
- 파일 최상단 제목(`# CLAUDE.md Review`)은 최초 1회만 작성한다.
- 각 리뷰는 `---` 구분선으로 시작하여 파일 하단에 누적 추가한다.
- **이전 리뷰의 내용은 절대 삭제/수정하지 않는다.**
- 2회차 이상에서는 체크리스트 앞에 "이전 리뷰 피드백 해소 여부" 인용 블록을 배치하여, 이전 피드백이 어떻게 반영되었는지 명시한다.

#### 템플릿

```markdown
# CLAUDE.md Review

---

## 리뷰 일자: YYYY-MM-DD

> **이전 리뷰 피드백 해소 여부** (2회차 이상에서만 작성)
> - [항목명]: 해소됨/미해소. (근거)

### 체크리스트

#### 포함 여부 (가이드 섹션 4)
- [ ] 기술 스택과 버전
- [ ] 프로젝트 구조
- [ ] 핵심 명령어
- [ ] 아키텍처 철학(WHY)
- [ ] 사용하는 패턴
- [ ] 사용하지 않는 패턴
- [ ] 도메인 용어 매핑
- [ ] 테스트 방법
- [ ] Git 워크플로우
- [ ] Claude가 자주 틀리는 항목

#### 제외 여부 (가이드 섹션 5)
- [ ] 보안 정보 없음
- [ ] 스타일 규칙 없음
- [ ] 프레임워크 기본 지식 설명 없음
- [ ] 과도한 문서 없음
- [ ] 프로젝트 히스토리 없음
- [ ] 태스크 한정 지시사항 없음

#### 길이 및 구조 (가이드 섹션 6)
- [ ] 300줄 이하
- [ ] 상세 내용은 파일 참조 방식으로 분리
- [ ] @-import 직접 임베드 없음

#### Anti-Pattern (가이드 섹션 8)
- [ ] 금지 항목에 대안 명시
- [ ] 매뉴얼 형태가 아닌 핵심 교정 중심
- [ ] 스타일 규칙은 결정론적 도구로 위임

### 문제점

(미통과 항목에 대해 가이드의 어느 기준(섹션 번호)에 근거하는지 명시하고, 현재 CLAUDE.md의 구체적인 문제를 기술)

#### 1. [문제 항목명] (가이드 섹션 N 근거)
- 현황: (현재 CLAUDE.md에서 해당 부분의 상태)
- 문제: (왜 문제인지)

### 개선점

(문제점별 구체적인 개선 방향과 예시 제시)

#### 1. [문제 항목명]
- 개선 방향: ...
- 예시:
  ```markdown
  (개선 후 CLAUDE.md에 들어갈 내용 예시)
  ```

### 종합 판정: **양호** / **개선 필요**
```

- 통과한 항목은 `[x]`, 미통과 항목은 `[ ]`로 표기
- 미통과 항목이 하나라도 있으면 **개선 필요**로 판정
