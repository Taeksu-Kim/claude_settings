# Code Review Workflow

PDCA 작업 단위(task_id)의 산출물과 코드를 기반으로 코드 리뷰를 수행한다.
본 워크플로우는 코드 품질 관리의 최종 게이트로서, 코드 품질과 설계를 종합적으로 검증하고 커밋 가능 여부를 판단한다.

## 산출물 경로

```
claude_task/<task_id>/review/
└── review.md
```

## 리뷰 절차

### 1. 컨텍스트 파악
- `claude_task/<task_id>/plan/plan.md` — 요청 의도와 설계 방향
- `claude_task/<task_id>/do/do.md` — 테스트 설계 의도
- `claude_task/<task_id>/check/check.md` — 점검 결과 및 수정 사항
- `claude_task/<task_id>/act/act.md` — 최종 구현 내용

### 2. 코드 리뷰 관점
다음 관점에서 종합적으로 리뷰한다:

#### 코드 품질
- **메모리 효율성**: 불필요한 복사, 과도한 메모리 사용이 없는가
- **변수명/함수명**: 의도가 명확하고 일관된 네이밍인가
- **오버 엔지니어링 방지**: 현재 불필요한 추상화나 기능이 포함되어 있지 않은가
- **미사용 코드**: 사용되지 않는 import, 변수, 함수가 남아 있지 않은가
- **에지 케이스**: 경계값, 예외 상황이 적절히 처리되는가

#### 설계 및 구조
- **설계 적합성**: Plan의 접근 방식과 실제 구현이 일치하는가
- **근본적 문제 해결**: 증상이 아닌 올바른 설계와 근본 원인을 해결하는 방향인가
- **책임 분리**: 함수/모듈이 단일 책임을 갖는가
- **의존성**: 불필요한 의존이나 순환 의존이 없는가
- **확장성**: 향후 변경에 유연한 구조인가 (단, 오버 엔지니어링은 지양)

#### 가독성 및 일관성
- **가독성**: 코드 흐름이 자연스럽고 이해하기 쉬운가
- **일관성**: 프로젝트의 기존 컨벤션과 일치하는가

### 3. 결과 기록
`review.md`에 다음 형식으로 기록한다.

#### 누적 기록 규칙
- 파일 최상단 제목(`# Code Review - <task_id>`)은 최초 1회만 작성한다.
- 각 사이클은 `---` 구분선으로 시작하여 파일 하단에 누적 추가한다.
- **이전 사이클의 내용은 절대 삭제/수정하지 않는다.**
- 2회차 이상에서는 체크리스트 앞에 "이전 사이클 피드백 해소 여부" 인용 블록을 배치하여, 이전 피드백이 어떻게 반영되었는지 명시한다.

#### 템플릿

```markdown
# Code Review - <task_id>

---

## 사이클 1/3

### 리뷰 대상 파일
- path/to/file1.py
- path/to/file2.py

### 체크리스트

#### 코드 품질
- [ ] 메모리 효율성: 불필요한 복사, 과도한 메모리 사용 없음
- [ ] 변수명/함수명: 의도가 명확하고 일관된 네이밍
- [ ] 오버 엔지니어링 방지: 불필요한 추상화나 기능 없음
- [ ] 미사용 코드: 사용되지 않는 import, 변수, 함수 없음
- [ ] 에지 케이스: 경계값, 예외 상황 적절히 처리

#### 설계 및 구조
- [ ] 설계 적합성: Plan의 접근 방식과 실제 구현 일치
- [ ] 근본적 문제 해결: 올바른 설계와 근본 원인 해결
- [ ] 책임 분리: 함수/모듈이 단일 책임
- [ ] 의존성: 불필요한 의존이나 순환 의존 없음
- [ ] 확장성: 향후 변경에 유연한 구조

#### 가독성 및 일관성
- [ ] 가독성: 코드 흐름이 자연스럽고 이해하기 쉬움
- [ ] 일관성: 프로젝트 기존 컨벤션과 일치

### 피드백
(미통과 항목에 대해 구체적인 코드 위치와 개선 방향 기술)

### 종합 판정: **승인(커밋 가능)** / **수정 필요**

---

## 사이클 2/3

> **이전 사이클 피드백 해소 여부**
> - [항목명]: 해소됨/미해소. (근거)

### 리뷰 대상 파일
(변경된 파일 목록)

### 체크리스트
(동일 체크리스트)

### 피드백
(해당 사이클의 피드백)

### 종합 판정: **승인(커밋 가능)** / **수정 필요**
```

- 통과한 항목은 `[x]`, 미통과 항목은 `[ ]`로 표기
- 미통과 항목이 하나라도 있으면 **수정 필요**로 판정
